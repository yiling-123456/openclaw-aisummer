# 消融草稿（Day3 · 样本轨迹）

- **变量**：`system-prompt`（有 / 无），其余（任务集 `eval/tasks.py`、模型 `deepseek-chat`）固定
- **结果**：
  - **有提示词**：成功率 = **1.00** - **无提示词**：成功率 = **0.00**
  - **token**：有提示词时消耗的输入 token 更高（因为注入了工具说明和行为准则）
- **归因**：无 `system-prompt` 时，agent 失去了系统赋予的角色、能力声明和指令防线，根本不知道 `<tool_call>` 的约定，导致它退化成普通聊天模型，从而“从不调用工具”，在邮件分类和本地文件读取任务中引发全盘失败。
- **局限**：目前样本量太小（各 2 ~ 3 条）、样本是人工构造的；D4 起将启用真实轨迹，届时每组将多次运行取均值后再下最终结论。
